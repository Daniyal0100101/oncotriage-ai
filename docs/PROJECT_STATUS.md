## What Still Needs to Happen Before Scientific Credibility Claims

1. **Real data training** — The RF ranking head is currently trained on synthetic ClinVar-like data generated by `build_synthetic_training_data()`. It must be retrained on real Evo 2 scores for known ClinVar variants.
2. **Benchmark evaluation** — Review priority scores must be compared against ClinVar ground-truth pathogenicity labels on a held-out test set and reported with AUC and calibration metrics.
3. **Scoring path validation** — The generate-proxy scoring approach (single-token logits) should be benchmarked against direct forward-pass logits on a private NIM deployment to quantify the approximation error.
4. **Gene coverage expansion** — Only BRCA1 and BRCA2 are currently targeted. Other hereditary cancer genes (MLH1, MSH2, PALB2, etc.) are out of scope.
5. **External review** — Code and methodology should be reviewed by a domain expert (genetic counselor or computational biologist) before any institutional use.

---

## Inference Architecture Note

The hosted NVIDIA NIM Evo 2 API (`health.api.nvidia.com`) does not expose the `/forward` logits endpoint. The scoring path currently uses the `/generate` endpoint with `enable_logits=True` and `num_tokens=1` to extract a single-token log-likelihood proxy. This is a functional approximation — not a direct per-residue logit readout. The `EVO2_USE_FORWARD_PASS` flag enables the true forward path for private NIM deployments only.
